구글 크롬, 사용자 동의 없이 4GB AI 모델을 기기에 자동 설치

구글 크롬이 사용자에게 알리거나 동의를 구하지 않고 Gemini Nano라는 4GB 규모의 AI 모델을 로컬 기기에 자동으로 다운로드·설치하고 있다. 브라우저 내장 AI 기능(번역, 요약 등)을 위한 것으로 보이나, 저장 공간 점유와 프라이버시 측면에서 문제가 제기된다. 사용자가 명시적으로 비활성화하지 않으면 기본적으로 설치되며, 설정 메뉴 깊숙이 숨겨져 있어 대부분 인지하지 못한다. 디스크 용량이 제한된 환경에서는 실질적 피해가 될 수 있으며, 동의 없는 설치라는 점에서 논란이 확산 중이다.

기업 IT 관리자는 크롬 정책(chrome://flags 또는 그룹 정책)에서 Gemini Nano 자동 다운로드를 비활성화하여 불필요한 디스크 사용과 네트워크 트래픽을 차단할 수 있다. 개인 사용자도 chrome://settings에서 해당 기능을 끄고 이미 설치된 모델 파일을 삭제해 공간을 확보할 수 있다.

LLM에 대해 이야기해 봅시다

저자는 Fred Brooks의 '은탄환은 없다' 프레임워크를 인용하며 LLM 코딩 도구가 소프트웨어 개발을 혁신할 수 없다고 주장한다. DORA 2025 보고서에 따르면 AI 도구가 처리량은 높이지만 '전달 불안정성'을 증가시키고, CircleCI 데이터는 복구 시간 25% 증가와 배포 실패 30% 증가를 보여준다. METR 연구에서는 개발자가 20% 속도 향상을 느꼈지만 실제로는 오히려 느려졌으며, 자기보고 편향 문제를 지적한다. 코딩은 개발 업무의 17% 미만이므로 근본적(essential) 어려움은 LLM으로 해결되지 않으며, 기존 개발 기본기가 더 중요하다고 결론짓는다.

팀에 AI 코딩 도구 도입 시 DORA/CircleCI 지표를 근거로 '속도 향상 vs 장애 복구 시간 증가' 트레이드오프를 경영진에게 설명하는 데 활용할 수 있다. 또한 LLM 생성 코드에 대한 리뷰·테스트 프로세스 강화의 필요성을 정당화하는 근거로 사용 가능하다.

Claude Agent SDK 기반 에이전트에 웹 브라우징 기능을 추가하여 자동 리서치, 폼 작성, 데이터 수집 등의 워크플로우를 구축할 때 활용 가능.
